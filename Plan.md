# 중계 플랫폼
## 플랫폼의 목적
  * 플랫폼은 중계자의 역할을 수행하며
  * 판매자와 구매자로 이루어지며 한 사용자가 판매자이면서 구매자일 수도 있음.
  * 판매자는 일반 상품이 아니라 서비스를 판매하는 판매자이며
    * 이를 위해 자신의 서비스에 대한 포트폴리오 형태의 전문가임을 알려주기 위한 프로필
     각종 서비스의 형태를 알려주기 위한 블로그 형태의 포트폴리오 (사진, 동영상, 텍스트 추가 가능)
     을 게시할 수 있어야 함. 
  * 구매자는 서비스의 요청을 게시할 수 있으며 이에 역 경매형식으로 다양한 판매자가 견적을 제시할 수 있음.
  * 구매자는 서비스 요청 작성 시 텍스트 이미지 를 입력 할 수 있고 첨부파일도 등록할 수 있음.
  * 구매자는 종료시간 이전에 선택을 해야하며 후보 업체의 견적을 오픈할 수 있고 최종 선택이 가능함. 
## 로그인 
  * 로그인은 자체 로그인과 소셜로그인을 할 수 있음 (네이버, 카카오톡, 구글, 애플)  
  * 사용자는 판매자와 구매자 기능 모두 동시 사용 가능함. 
## 기술 스택
  * 프론트엔드: React (Vite) + TypeScript + Tailwind CSS
    -  Font Pretended, Color Thema: Orange 계열
    - 화면은 반응형으로 제공
  * 백엔드: Python + FastAPI + SQLAlchemy + Pydantic 
  * 도커: Postgresql은 도커로 운영 프론트와 백엔드는 도커없이 운영
위의 내용을 이용하여 상세 설계하여 DetailedPlan.md를 만들것 (필요한 사항은 질문할 것)
